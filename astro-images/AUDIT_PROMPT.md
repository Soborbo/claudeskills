# Astro Images Audit Checklist (v3.1)

Run this to verify an existing site complies with the astro-images skill. **Read-only:**
this is an audit — report **PASS / FAIL / N-A** with file:line (or byte) evidence; fix via
`REMEDIATION_PROMPT.md`. **Authority:** `SKILL.md` + `rules.json` (v3.1). If a grep here
ever disagrees with SKILL.md's Validation block, SKILL.md wins. **Flag, don't guess** —
the real LCP, ambiguous crops/faces, and unknown alt keyword/locality need human judgment.

---

## 0. Orient (read before judging anything)
Read `package.json` (build + image scripts), `astro.config.*`, `src/layouts/*.astro`, every
image component that exists (`Picture`, `ArtPicture`, `FixedImage`, plus any bespoke
`OptimizedPicture`/`PictureImg`/…), the preprocessor (`scripts/optimize-images.mjs` or
similar), `public/robots.txt`, and 3–4 representative pages. List **every** image pipeline in
use and which files feed each.

---

## 1. File structure & source masters
- [ ] Content images in `/src/assets/` (Astro-processed), OR in `/public/` **only** when
      generated by `optimize-images.mjs`.
- [ ] `/public/` image sets are **100% script-generated** — no hand-placed files in `public/img` (IMG_OUT).
- [ ] Source masters live in `IMG_SRC` (`src/assets/preprocess`) and are committed to the
      (private) repo; `IMG_OUT` is disposable/reproducible. (`--clean` wipes IMG_OUT only and
      is safe only with full source coverage.)

```bash
# Raw images dropped straight into /public/ without a preprocessor = suspicious
find public -type f \( -name "*.jpg" -o -name "*.jpeg" -o -name "*.png" -o -name "*.webp" -o -name "*.avif" \) 2>/dev/null | grep -v "/og/" | head
# If public/img exists, every slug must have a master in IMG_SRC (else --clean would lose it)
```

---

## 2. Components & responsive output
- [ ] Content images use `<Picture pattern>` (or `ArtPicture`) — not raw raster `<img>`.
- [ ] Every `<Picture>` / `ArtPicture` has `alt` (`alt=""` only if decorative).
- [ ] Every content image is responsive (srcset + sizes); **no single-size content images**.
- [ ] **480w** present in every srcset.
- [ ] Custom `widths`/`sizes` only as intentional overrides; the pattern handles the rest.

```bash
# <Picture> without a pattern (allowed only if HALF default is intentional)
grep -rn "<Picture" src --include="*.astro" | grep -v "pattern="
# <Picture> missing alt
grep -rn "<Picture" src --include="*.astro" | grep -v "alt="
# 480w must appear in built srcsets
grep -rl "480w" dist --include="*.html" | wc -l
```

---

## 3. Per-format quality — AVIF must be SMALLER than WebP  *(v3.1 headline)*
- [ ] AVIF encoded at a **lower** quality than WebP (per-format, never one shared value).
      `QUALITY` in `image-patterns.ts`: avif 50, webp 60, jpg 62; AVIF `effort` 5 in the script.
- [ ] AVIF output is **actually smaller** than WebP at the same width (spot-check a hero).
- [ ] No single shared quality for all formats.

```bash
# Spot-check one hero at ~1600w — AVIF must be < WebP (replace <hero> with a real basename)
ls -l dist/_astro/<hero>*1600*.avif dist/_astro/<hero>*1600*.webp
# Numeric quality={NN} still works (back-compat shim) but should migrate to the per-format object
grep -rEn 'quality=\{[0-9]+\}' src --include="*.astro"
```
> If any AVIF ≥ WebP → **FAIL**: lower AVIF quality and regen (`npm run images:force`, or `--force`).

---

## 4. Alpha-aware fallback (JPG opaque / PNG transparent)
- [ ] Fallback is **JPG for opaque**, **PNG only for transparent** (logos/icons/alpha).
- [ ] No opaque-PNG photos in the output.

```bash
find dist -name "*.png" -not -path "*/icons/*" -not -path "*/svg/*"   # opaque PNG photos = FAIL
```

---

## 5. Pattern selection (must match layout)

| Layout | Correct pattern |
|--------|-----------------|
| Full-bleed hero | `FULL` |
| Split 66/33, 60/40 (image side) | `TWO_THIRDS`, `LARGE` |
| Split 50/50, checkerboard | `HALF` |
| Card at 50% with max-height | `HALF_CARD` |
| Text-dominant split (40/60) | `SMALL` |
| 3-col grid, standing person | `THIRD` |
| 4-col team grid | `QUARTER` |
| 5-col icons | `FIFTH` |
| 6-col logos | `SIXTH` |
| Different image/crop per breakpoint | `ArtPicture` (see §6) |
| Logo, avatar, icon | `FixedImage` |

- [ ] Each `pattern` matches the actual CSS layout; unknown → `HALF`.
- [ ] Logo/avatar/icon → `FixedImage` (not `<Picture>`).

---

## 6. Art direction — one download per slot
- [ ] Different image/crop per breakpoint uses **`ArtPicture`** (`<source media>` → exactly one download).
- [ ] Same image, different size only → **`<Picture>`** (NOT `ArtPicture` — it is already one download).
- [ ] **Never** two `<img>` + `hidden md:block` (a `display:none` image is still downloaded → double load).
- [ ] Checkerboard: ONE image reordered via CSS `order`; image **first in DOM** (mobile: on top).

```bash
# Review for the double-load anti-pattern
grep -rn "hidden md:block\|md:hidden" src --include="*.astro" | grep -iE "<img|Picture"
```

---

## 7. LCP / priority / preload  *(changed in v3.1)*
- [ ] Exactly **ONE** `lcp` per page; `lcp` not inside a loop/map.
- [ ] Hero uses `lcp` (loading=eager + fetchpriority=high); below-fold lazy; 2–3 above-fold use `aboveFold`.
- [ ] **No fixed-width image preload** (`preloadImage="/img/x-480w.avif"` and the
      `<link rel=preload as=image href=…>` it emits) — harmful, removed in v3.1.
- [ ] Only a genuinely late-discovered hero (CSS background / JS-injected) may use a
      **responsive** preload (`imagesrcset` + `imagesizes` matching the picture exactly).

```bash
grep -rn "preloadImage" src --include="*.astro"                                            # should be empty
grep -rn 'rel="preload"' dist --include="*.html" | grep 'as="image"' | grep -v "imagesrcset"  # fixed-width preload = FAIL
grep -rn "fetchpriority" src --include="*.astro"                                           # review: one LCP per page
grep -rn "lcp" src --include="*.astro" | grep -E "\.(map|forEach)\("                       # lcp in a loop = FAIL
```

---

## 8. Forbidden output formats
- [ ] No `<Picture>` for SVG (raw `<img>` with width/height from viewBox).
- [ ] No animated GIF/APNG/animated WebP (use `<video>`).
- [ ] No CSS `background-image` for LCP / content images.
- [ ] No upscaling beyond source width.

```bash
grep -rn "<Picture" src --include="*.astro" | grep -iE "\.svg|\.gif"
```

---

## 9. Raw `<img>` rules
Raw `<img>` allowed **only** for: `FixedImage` output, SVGs, external URLs.

- [ ] No raw raster content `<img>` (PNG/JPG without the pipeline).
- [ ] SVG `<img>`: explicit `width`/`height`. External: `width`, `height`, `loading="lazy"`, `decoding="async"`.
- [ ] `width`/`height` = delivered dimensions, not source.

```bash
grep -rEn '<img[^>]+\.(png|jpe?g)' src --include="*.astro" | grep -v "FixedImage"   # raster content <img> = FAIL
```

---

## 10. Source image sizes (minimums)

| Pattern | Min source width |
|---------|------------------|
| FULL | 2560 |
| TWO_THIRDS | 2048 |
| LARGE | 1920 |
| HALF | 1600 |
| HALF_CARD | 1280 |
| SMALL / THIRD | 1280 |
| QUARTER | 960 |
| FIFTH | 768 |
| SIXTH | 640 |

```bash
# List source dimensions (requires ImageMagick `identify`)
find src/assets -type f \( -name "*.jpg" -o -name "*.png" -o -name "*.webp" \) -exec identify -format "%f: %wx%h\n" {} \;
```
- [ ] No source smaller than its pattern minimum. **FULL/LCP undersized = ERROR**; others = WARNING (widths capped, flag for replacement).

---

## 11. Image SEO — filenames + alt  *(new in v3.1)*
- [ ] **Source filenames** = kebab-case keyword-locality slugs (Astro keeps the basename in the URL).
- [ ] No `IMG_*` / `DSC_*`, spaces, accents, or generic `hero.jpg` / `image.jpg` / `photo.jpg`.
- [ ] **Alt**: accurate + natural keyword, **no stuffing**, no "image of"/"photo of", ≤ ~125 chars, decorative `alt=""`.
- [ ] Unknown content / keyword / locality → **FLAG** (don't invent).
- [ ] `<figure>`/`<figcaption>` where it adds real context; hero in `ImageObject`/schema (coordinate with `schema-audit`).

```bash
# SEO-poor source filenames
find src/assets -type f \( -iname "IMG_*" -o -iname "DSC_*" -o -iname "hero.*" -o -iname "image.*" -o -iname "photo.*" \)
find src/assets -type f -name "* *"                       # spaces in filenames
# Generic / "image of" alt text
grep -rEin 'alt="(image|photo|picture)"' src --include="*.astro"
grep -rEin 'alt="(image|photo) of ' src --include="*.astro"
```

---

## 12. OG / social images  *(og:image:type added in v3.1)*
- [ ] OG images are **JPG** (social platforms don't render AVIF/WebP → blank preview).
- [ ] `og:image` points at a **generated, existing** file (no 404).
- [ ] **`og:image:type` present and matching the file** — `image/jpeg` for a JPG OG,
      `image/png` for pages still on the `og-default.png` fallback.
- [ ] `og:image:width`/`height` set; `twitter:image` (JPG); 5 schema variants where used.

```bash
grep -rEn 'ogImage\s*[:=].*\.(webp|avif)' src --include="*.astro"        # WebP/AVIF OG = FAIL
grep -rn 'og:image:type' src --include="*.astro" || echo "MISSING og:image:type"
```

---

## 13. Face focus
- [ ] People images have `object-position` (e.g. `class="object-[center_20%]"`); non-person → center;
      ambiguous (multiple people, subject at edge, unclear) → **FLAG** (discuss with user).

---

## 14. robots.txt
- [ ] `robots.txt` does **not** `Disallow: /_astro/` (or the image dir) — else Google Images can't index.

```bash
grep -n "_astro" public/robots.txt 2>/dev/null && echo "WARNING: ensure robots does not Disallow /_astro/"
```

---

## 15. Astro / adapter config
- [ ] `output: 'static'` (or `'server'` if API routes/forms).
- [ ] adapter `imageService: 'compile'` (Sharp at **build** time) — works with `static` OR `server`.
- [ ] `image.service.entrypoint: 'astro/assets/services/sharp'`.

```bash
grep -nE "output:|imageService|entrypoint" astro.config.* 2>/dev/null
```

---

## 16. Preprocessor (`/public/` sets) — incremental & --clean safety
- [ ] `optimize-images.mjs`: AVIF q50 + effort 5, WebP 60, JPG 62; alpha-aware fallback.
- [ ] Incremental by default; `--force` regenerates in place; `--clean` wipes `IMG_OUT` only.
- [ ] `--clean` used **only** with full source coverage in `IMG_SRC` (else un-sourced images are lost).
- [ ] `package.json` has `images` / `images:force` / `images:clean`.

---

## 17. Build & browser verification
```bash
npm run build       # must pass
find dist/_astro -name "*.avif" -o -name "*.webp" | head
find dist -name "*.png" -not -path "*/icons/*" -not -path "*/svg/*"   # opaque PNG = FAIL
grep -rl "480w" dist --include="*.html" | wc -l
```
- [ ] Build passes; AVIF + WebP generated; **AVIF < WebP**; no opaque-PNG; 480w in every srcset.
- [ ] DevTools → Network → Img: exactly **ONE** image per `<Picture>`/`ArtPicture`; correct format
      (AVIF in Chrome/Firefox, WebP fallback); not oversized for viewport; no duplicate requests.

---

## 18. Lighthouse
- [ ] No "Properly size images" / "Serve images in next-gen formats" / "Efficiently encode images" warnings.
- [ ] LCP image loads within 2.5s; no image CLS (width/height or aspect-ratio set).

## If the audit fails
1. Identify the failed rule + offending `file:line`.
2. Fix per `SKILL.md`, or run `REMEDIATION_PROMPT.md` for a full guided fix.
3. Re-run the failed checks **plus** the Validation block in `SKILL.md`.
4. Full audit + `npm run build` pass before deploy.
