# Astro Images — Acceptance Tests (v3.1)

60 discrete, falsifiable tests an image (or image pipeline) managed by this skill must pass.
Derived from `SKILL.md` + `rules.json` (v3.1). **Authority:** if a check here disagrees with
SKILL.md's Validation block, SKILL.md wins. This is a **spec**, not a fixer — failures are
remediated via `REMEDIATION_PROMPT.md`. Audit walkthrough lives in `AUDIT_PROMPT.md`.

**Legend** — 🟢 automatable (grep/command, run from the site repo root) · 🔵 build-time
(after `npm run build`) · ⚪ manual / human judgment (flag, don't guess).

Pass = the stated condition holds. For 🟢 "→ empty" tests, a non-empty result is a FAIL.

---

## A. File structure & source masters
- **T01** ⚪ Every content image is sourced from `/src/assets/` (Astro-processed) or `/public/` **only** when generated by `optimize-images.mjs`.
- **T02** 🟢 `public/img` (IMG_OUT) holds only script-named files. → `find public/img -type f 2>/dev/null | grep -vE -- '-[0-9]+w\.(avif|webp|jpg|png)$'` is empty.
- **T03** ⚪ Every image in IMG_OUT has a master in IMG_SRC (`src/assets/preprocess`).
- **T04** 🟢 Source masters are committed, not ignored. → `git check-ignore src/assets/preprocess` is empty AND `git ls-files src/assets/preprocess` is non-empty.
- **T05** ⚪ Exactly one master per image (no duplicate sources).

## B. Component usage
- **T06** 🟢 No raw raster content `<img>`. → `grep -rEn '<img[^>]+\.(png|jpe?g)' src --include="*.astro" | grep -v FixedImage` is empty.
- **T07** 🟢 Every `<Picture>` has a `pattern` (or an intentional HALF default). → review `grep -rn "<Picture" src --include="*.astro" | grep -v "pattern="`.
- **T08** 🟢 Every `<Picture>`/`ArtPicture` has `alt` (`alt=""` only if decorative). → `grep -rn "<Picture" src --include="*.astro" | grep -v "alt="` is empty.
- **T09** ⚪ Logos / avatars / icons use `FixedImage`, not `<Picture>`.
- **T10** 🟢 SVG never goes through `<Picture>`. → `grep -rn "<Picture" src --include="*.astro" | grep -i "\.svg"` is empty.
- **T11** ⚪ Custom `widths`/`sizes` props appear only as deliberate overrides; otherwise the pattern drives them.

## C. Pattern selection & source size
- **T12** ⚪ Each `pattern` matches the actual CSS layout (FULL hero, HALF 50/50, QUARTER 4-col, …).
- **T13** ⚪ Unknown / ambiguous layout falls back to `HALF`.
- **T14** 🔵 Every srcset includes **480w** (and every pattern's `widths` array contains 480). → `grep -rl "480w" dist --include="*.html"` is non-empty.
- **T15** 🟢 No source smaller than its pattern's `minSourceWidth` (else widths capped + flagged). → `identify` dimensions vs. the Source Minimums table.
- **T16** ⚪ A FULL/LCP image is never undersized (FULL source ≥ 2560px) — **ERROR** if violated.
- **T17** 🔵 `width`/`height` are the **delivered** dimensions (not the source) → no CLS.

## D. Per-format quality (v3.1 headline)
- **T18** 🟢 AVIF quality is lower than WebP. → `image-patterns.ts` `QUALITY.avif (50) < QUALITY.webp (60)`.
- **T19** ⚪ Never one shared quality value across formats.
- **T20** 🔵 For a representative hero, AVIF bytes < WebP bytes at the same width. → `ls -l dist/_astro/<hero>*1600*.avif dist/_astro/<hero>*1600*.webp`.
- **T21** 🟢 The manual pipeline uses AVIF `effort` ≥ 5. → `optimize-images.mjs` `AVIF_EFFORT`.
- **T22** ⚪ Heavy-hero AVIF override uses the object form (`quality={{ avif: 46 }}`), not a bare number.

## E. Formats & alpha-aware fallback
- **T23** 🔵 Three formats in order: `<source avif>` → `<source webp>` → `<img jpg|png>`.
- **T24** 🔵 Opaque sources fall back to **JPG**.
- **T25** 🔵 PNG fallback **only** for transparent (alpha) sources.
- **T26** 🟢 No opaque-PNG photos in output. → `find dist -name "*.png" -not -path "*/icons/*" -not -path "*/svg/*"` is empty.
- **T27** 🟢 No GIF / APNG / animated WebP (use `<video>`).

## F. Responsive output
- **T28** 🔵 No single-size content image — every content image's srcset has ≥ 2 entries.
- **T29** 🔵 Every responsive image has `sizes`, matching the pattern's `sizes`.
- **T30** ⚪ Lazy images may use `sizes="auto"`; eager images carry explicit `sizes`.

## G. LCP / priority / preload
- **T31** 🟢 Exactly one `lcp` per page. → per-page count of `lcp` props ≤ 1.
- **T32** 🟢 `lcp` is not inside a loop. → `grep -rn "lcp" src --include="*.astro" | grep -E "\.(map|forEach)\("` is empty.
- **T33** 🔵 The hero (`lcp`) `<img>` has `loading="eager"` **and** `fetchpriority="high"`.
- **T34** ⚪ Below-fold images are lazy; never eager.
- **T35** 🟢 No fixed-width image preload. → `grep -rn "preloadImage" src --include="*.astro"` is empty AND `grep -rn 'rel="preload"' dist --include="*.html" | grep 'as="image"' | grep -v imagesrcset` is empty.
- **T36** ⚪ A late-discovered hero, if preloaded, uses a **responsive** preload (`imagesrcset` + `imagesizes`) matching the picture.

## H. Art direction / one download per slot
- **T37** ⚪ Same image at different sizes uses `<Picture>` (already one download), not `ArtPicture`.
- **T38** ⚪/🔵 Different image/crop per breakpoint uses `ArtPicture` (`<source media>`) → the browser downloads **exactly one** (verify in DevTools → Network).
- **T39** 🟢 No two `<img>` + `hidden md:block` / `md:hidden` for art direction. → review `grep -rn "hidden md:block\|md:hidden" src --include="*.astro" | grep -iE "<img|Picture"`.
- **T40** ⚪ Checkerboard: one image reordered via CSS `order`, image **first in DOM** (mobile on top); no `flex-col-reverse` / `order-first` on mobile.

## I. OG / social images
- **T41** 🟢 OG image is JPG (not AVIF/WebP). → `grep -rEn 'ogImage\s*[:=].*\.(webp|avif)' src --include="*.astro"` is empty.
- **T42** ⚪ `og:image` points at an existing, generated file (no 404).
- **T43** 🟢 `og:image:type` is present. → `grep -rn 'og:image:type' src --include="*.astro"`.
- **T44** ⚪ `og:image:type` matches the actual file (`image/jpeg` for JPG, `image/png` for the `og-default.png` fallback).
- **T45** 🟢 `og:image:width` and `og:image:height` are set. → `grep -rn 'og:image:width' src --include="*.astro"`.
- **T46** ⚪ `twitter:image` present and JPG; 5 schema variants where used.

## J. Image SEO
- **T47** ⚪ Source filenames are kebab-case keyword-locality slugs (Astro keeps the basename in the URL).
- **T48** 🟢 No `IMG_*` / `DSC_*` source names. → `find src/assets -type f \( -iname "IMG_*" -o -iname "DSC_*" \)` is empty.
- **T49** 🟢 No spaces / accents in source filenames. → `find src/assets -type f -name "* *"` is empty.
- **T50** 🟢 No generic source names. → `find src/assets -type f \( -iname "hero.*" -o -iname "image.*" -o -iname "photo.*" \)` is empty.
- **T51** 🟢/⚪ Alt is descriptive + natural, no stuffing, no "image of"/"photo of", ≤ ~125 chars. → `grep -rEin 'alt="(image|photo|picture)"' src --include="*.astro"` is empty + manual quality.
- **T52** ⚪ Decorative images use `alt=""`; unknown keyword/locality is **flagged**, not invented.

## K. Face focus
- **T53** ⚪ People images set `object-position` (e.g. `object-[center_20%]`); non-person → center.
- **T54** ⚪ Ambiguous focal points were flagged/asked, not guessed.

## L. Raw `<img>` rules
- **T55** 🟢 Raw `<img>` only for FixedImage output, SVG, or external URLs. → covered by T06; SVG/external reviewed.
- **T56** ⚪ SVG `<img>` has explicit `width`/`height`; external `<img>` has `width`+`height`+`loading="lazy"`+`decoding="async"`.

## M. Indexability & config
- **T57** 🟢 `robots.txt` does not block `/_astro/`. → `grep -n "_astro" public/robots.txt` shows no `Disallow`.
- **T58** 🟢 Adapter config correct. → `grep -nE "output:|imageService|entrypoint" astro.config.*` shows `imageService: 'compile'` + sharp entrypoint.

## N. Pipeline & build
- **T59** 🟢 Preprocessor is incremental (mtime skip), `--force` regenerates in place, `--clean` wipes IMG_OUT only and exits first if IMG_SRC is empty (guard). → inspect `optimize-images.mjs`.
- **T60** 🔵 `npm run build` passes, emits AVIF+WebP variants, 480w in every srcset, no opaque-PNG.
